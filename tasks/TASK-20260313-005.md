## 📋 任务描述

为任务公告板首页的统计卡片增加筛选页面，让用户可以点击"进行中"、"等待中"、"已完成"查看对应状态的任务列表。

**问题**: 主页统计卡片中，只有"阻塞中"有链接到 blocked.html，其他状态（进行中、等待中、已完成）没有对应的筛选页面。

**用户原话**: 
> "主页里，阻塞中能链接到阻塞的任务页面。进行中，等待中，已完成，没有链接对应的页面。"

## 📊 功能需求

### 1. 进行中页面 (progress.html)

**URL**: `/task-board/progress.html`

**功能**:
- 显示所有 `IN_PROGRESS` 状态的任务
- 任务卡片包含：任务 ID、标题、执行者、进度条、预计完成时间
- 支持点击任务 ID 跳转到详情页
- 支持刷新按钮

**数据源**: `GET /api/tasks/blocked/all` → 筛选 `status === 'IN_PROGRESS'`

### 2. 等待中页面 (pending.html)

**URL**: `/task-board/pending.html`

**功能**:
- 显示所有 `PENDING` 和 `ASSIGNED` 状态的任务
- 任务卡片包含：任务 ID、标题、执行者、优先级
- 支持点击任务 ID 跳转到详情页

**数据源**: `GET /api/tasks/blocked/all` → 筛选 `status === 'PENDING' || status === 'ASSIGNED'`

### 3. 已完成页面 (completed.html)

**URL**: `/task-board/completed.html`

**功能**:
- 显示所有 `COMPLETED` 状态的任务
- 任务卡片包含：任务 ID、标题、执行者、完成时间
- 支持点击任务 ID 跳转到详情页

**数据源**: `GET /api/tasks/blocked/all` → 筛选 `status === 'COMPLETED'`

## 🔧 技术实现

### 1. 创建 HTML 页面

**文件**:
- `public/progress.html` - 进行中任务
- `public/pending.html` - 等待中任务
- `public/completed.html` - 已完成任务

### 2. 扩展 task-lists.js

**新增函数**:
```javascript
// 加载进行中任务
async function loadProgressTasks() {
  const data = await fetch('/task-board-api/api/tasks/blocked/all');
  const inProgress = data.data.all.filter(t => t.status === 'IN_PROGRESS');
  renderTaskList(inProgress, 'task-list');
}

// 加载等待中任务
async function loadPendingTasks() {
  const data = await fetch('/task-board-api/api/tasks/blocked/all');
  const pending = data.data.all.filter(t => 
    t.status === 'PENDING' || t.status === 'ASSIGNED'
  );
  renderTaskList(pending, 'task-list');
}

// 加载已完成任务
async function loadCompletedTasks() {
  const data = await fetch('/task-board-api/api/tasks/blocked/all');
  const completed = data.data.all.filter(t => t.status === 'COMPLETED');
  renderTaskList(completed, 'task-list');
}
```
