## 📊 设计输出

### 1. 评审状态机设计

**新增状态**:
- `PENDING_REVIEW` - 待评审
- `REVIEWING` - 评审中
- `REVIEW_APPROVED` - 评审通过
- `REVIEW_REJECTED` - 评审驳回

**状态流转**:
```
PENDING → PENDING_REVIEW → REVIEWING → REVIEW_APPROVED → IN_PROGRESS
                                        ↓
                                   REVIEW_REJECTED → PENDING_REVIEW
```

### 2. 评审页面设计规范

**URL**: `/task-board/review/TASK-XXX.html`

**页面元素**:
- 任务详情展示区
- 评审标准展示区
- 评审意见输入框
- 通过/驳回操作按钮
- 评审历史时间线

### 3. 评审 API 设计规范

**接口**:
```javascript
POST /api/tasks/:taskId/review/start
  - 开始评审
  - 状态：PENDING_REVIEW → REVIEWING

POST /api/tasks/:taskId/review/approve
  - 评审通过
  - 状态：REVIEWING → REVIEW_APPROVED
  - 参数：comments, reviewer

POST /api/tasks/:taskId/review/reject
  - 评审驳回
  - 状态：REVIEWING → REVIEW_REJECTED
  - 参数：reasons, reviewer, suggestions
```

### 4. 评审记录数据设计

**数据表**: `task_reviews`

**字段**:
- id
- task_id
- reviewer
- result (approved/rejected)
- comments
- reasons
- suggestions
- created_at
