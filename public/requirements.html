    • 项目概述

    项目名称

    OpenMOSS-CN - 多 Agent 监控与协作系统 (基于 OpenClaw)

    项目目标

    打造一个类似 OpenMOSS 的多 Agent 监控与协作系统，实现：
  • 自主任务认领 - Agent 唤醒时自主 claim 任务
  • 完整任务生命周期 - 从创建到完成的全流程管理
  • 实时监控面板 - 实时展示 Agent 状态和任务进度
  • 活动日志追踪 - 所有操作可追溯
  • 数据持久化 - 所有数据存储在数据库，方便展示和查询

    • 核心价值

    | 价值点 | 说明 | |--------|------| | 减少人工干预 | Agent 自主认领任务，无需手动分配 | | 提高协作效率 | 多 Agent 并行工作，自主协调 | | 质量保障 | 评审 + 打分 + 返工闭环 | | 可追溯性 | 完整活动日志，问题可追溯 | | 数据驱动 | 数据库存储，支持分析和展示 |

    需求背景

    当前痛点

  • 任务分配繁琐 - 需要手动指定执行者
  • 进度不透明 - 无法实时查看 Agent 工作状态
  • 缺少闭环 - 评审流程不完整，无打分机制
  • 数据分散 - 任务数据在文件中，难以统计展示
  • 无法追溯 - 没有活动日志，问题难排查

    • OpenMOSS 启发

    OpenMOSS 项目 (https://github.com/d9g/OpenMOSS) 提供了完整的解决方案： 核心特性:
  • ✅ 自主任务认领机制
  • ✅ 完整任务状态机
  • ✅ 评审打分闭环
  • ✅ 巡逻监控 Agent
  • ✅ 活动日志 Feed
  • ✅ SQLite 数据库存储
  • ✅ FastAPI 后端
  • ✅ Vue 3 前端
    • 可借鉴点:
  • 数据库锁机制防止并发冲突
  • 任务状态机管理生命周期
  • 活动日志记录所有操作
  • RESTful API 提供数据接口

    • 我们的优势

  • 已有基础 - 任务板系统已运行
  • 更多 Agent - 6 个角色 vs OpenMOSS 4 个
  • OpenClaw 技能 - 可插拔技能系统
  • 简洁架构 - 易于理解和维护

    核心功能

    功能 1: 自主任务认领 ⭐⭐⭐ (P0)

    需求描述: Agent 唤醒时能够自主认领任务，无需人工分配。 实现方案:
  • 数据库锁机制 - 使用 SQLite 事务锁防止并发冲突
  • 认领 API - POST /api/tasks/{id}/claim
  • 超时释放 - 30 分钟无进展自动释放
  • 任务选择 - 按优先级和角色匹配
    • 数据表: 
    CREATE TABLE tasks (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    priority TEXT DEFAULT 'P2',
    status TEXT DEFAULT 'AVAILABLE',  -- AVAILABLE/ASSIGNED/IN_PROGRESS/COMPLETED
    claimed_by TEXT,                   -- 认领者
    claimed_at TIMESTAMP,              -- 认领时间
    ...
);
    API: 
    POST /api/tasks/{task_id}/claim
Body: {"agent_id": "xiaoyun-dev"}
Response: {"success": true, "task": {...}}
    验收标准:
  • [ ] Agent 唤醒时能自动认领任务
  • [ ] 多 Agent 不会认领同一任务
  • [ ] 认领超时自动释放
  • [ ] 活动日志正确记录

    • 功能 2: 任务状态机 ⭐⭐⭐ (P0)

    需求描述: 实现完整的任务状态流转，管理任务生命周期。 状态流转: 
    AVAILABLE → ASSIGNED → IN_PROGRESS → COMPLETED
                           ↓
                        BLOCKED → AVAILABLE (释放后)
                           
COMPLETED → REVIEW → APPROVED
              ↓
           REJECTED → IN_PROGRESS (返工)
    状态定义: | 状态 | 说明 | 转换条件 | |------|------|----------| | AVAILABLE | 可用 | 初始状态/释放后 | | ASSIGNED | 已分配 | 认领成功 | | IN_PROGRESS | 进行中 | Agent 开始执行 | | BLOCKED | 阻塞 | 遇到依赖/问题 | | REVIEW | 评审中 | 提交交付物 | | APPROVED | 已通过 | 评审通过 | | REJECTED | 已拒绝 | 评审不通过 | | COMPLETED | 已完成 | 最终状态 | 验收标准:
  • [ ] 状态流转正确
  • [ ] 状态变更有日志
  • [ ] 不允许非法流转

    Agent 唤醒逻辑

    const API_BASE = 'http://localhost:6565/api';

async function wakeUp() {
  console.log([${this.agentId}] 唤醒...);
  
  try {
    // 1. 获取可用任务
    const availableTasks = await this.getAvailableTasks();
    
    if (availableTasks.length === 0) {
      console.log([${this.agentId}] 无可用任务);
      return;
    }
    
    // 2. 选择适合的任务
    const suitableTask = this.selectTask(availableTasks);
    
    if (!suitableTask) {
      console.log([${this.agentId}] 没有适合的任务);
      return;
    }
    
    // 3. 尝试认领
    const claimResult = await this.claimTask(suitableTask.id);
    
    if (!claimResult.success) {
      console.log([${this.agentId}] 认领失败：${claimResult.error});
      return;
    }
    
    // 4. 执行任务
    console.log([${this.agentId}] 开始执行任务：${suitableTask.title});
    await this.executeTask(suitableTask);
    
    // 5. 提交交付物
    await this.submitDeliverable(suitableTask.id);
    
  } catch (error) {
    console.error([${this.agentId}] 唤醒失败:, error);
  }
}

async function getAvailableTasks() {
  const response = await fetch(${API_BASE}/tasks/available);
  const data = await response.json();
  
  if (data.success) {
    return data.data;
  } else {
    return [];
  }
}

function selectTask(tasks) {
  // 1. 按角色筛选
  const suitable = tasks.filter(task => 
    this.matchRole(task, this.role)
  );
  
  // 2. 按优先级排序
  suitable.sort((a, b) => {
    const priorityOrder = { 'P0': 0, 'P1': 1, 'P2': 2, 'P3': 3 };
    return priorityOrder[a.priority] - priorityOrder[b.priority];
  });
  
  // 3. 返回最高优先级的任务
  return suitable[0];
}

async function claimTask(taskId) {
  const response = await fetch(${API_BASE}/tasks/${taskId}/claim, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      agent_id: this.agentId
    })
  });
  
  return await response.json();
}
