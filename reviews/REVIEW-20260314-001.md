## 待决策问题回答

### 1. 角色模板粒度

**决策**: **中等粒度** (5-8 个核心角色)

**理由**:
- 太少 (2-3 个) → 无法满足多样化需求
- 太多 (15+ 个) → 维护成本高，用户选择困难
- 中等 (5-8 个) → 覆盖主要场景，易于维护

**v1.0 模板清单**:
1. `developer` - 功能开发
2. `tester` - 质量保障
3. `recorder` - 文档管理
4. `judge` - 质量评审
5. `dispatcher` - 任务调度

**v1.5 新增**:
6. `researcher` - 信息收集
7. `designer` - 产品设计

**v2.0 新增**:
8. `devops` - 运维部署

### 2. 继承范围

**决策**: **选择性继承** (用户配置，默认继承核心文件)

**默认继承**:
```yaml
inheritance:
  inherit_files:
    - SOUL.md        # 核心信念（保持团队一致性）
    - AGENTS.md      # 工作指南（标准化流程）
    - USER.md        # 用户信息（同一用户）
```

**默认不继承**:
```yaml
  override_files:
    - IDENTITY.md    # 身份信息（每个 Agent 独立）
    - HEARTBEAT.md   # 心跳任务（角色特定）
    - MEMORY.md      # 记忆文件（每个 Agent 独立）
    - TOOLS.md       # 工具配置（角色特定）
```

**理由**:
- SOUL.md/AGENTS.md → 保持团队文化和工作流程一致
- IDENTITY.md/HEARTBEAT.md → 每个角色有独特职责
- MEMORY.md → 每个 Agent 独立记忆，避免污染

### 3. 注册方式

**决策**: **自动注册 + 可选确认**

**方案**:
```bash
# 默认：自动注册
team-creator create team.yaml

# 交互模式：需要用户确认
team-creator create team.yaml --interactive

# 预览模式：不实际注册
team-creator create team.yaml --dry-run
```

**理由**:
- 默认自动注册 → 提升效率，符合"5 分钟创建团队"的目标
- 提供 `--interactive` → 满足谨慎用户需求
- 提供 `--dry-run` → 允许预览，降低误操作风险

## 问题和建议

### 严重问题 (必须修复)

**无** - 方案设计完善，无严重问题

### 改进建议 (推荐优化)

1. **配置冲突检测** (优先级：高)
   - **问题**: 注册器未检查 Agent ID 是否已存在
   - **建议**: 在 `registrar.js` 中添加冲突检测
   ```javascript
   if (config.agents[agentConfig.id]) {
     throw new Error(`Agent "${agentConfig.id}" 已存在，请使用不同 ID 或先删除`);
   }
   ```

2. **Gateway 重启故障排查** (优先级：中)
   - **问题**: Gateway 重启失败时用户可能不知所措
   - **建议**: 在 README 中添加故障排查章节
   ```markdown
   ## 故障排查
   
   ### Gateway 重启失败
   1. 检查 Gateway 状态：`openclaw gateway status`
   2. 查看日志：`tail -f ~/.openclaw/logs/gateway.log`
   3. 手动重启：`openclaw gateway restart`
   ```

3. **团队克隆冲突处理** (优先级：中)
   - **问题**: 克隆时目标目录已存在如何处理？
   - **建议**: 提供三种策略
   ```bash
   team-creator clone source.yaml --name "new-team" --conflict-strategy "fail"  # 默认：失败
   team-creator clone source.yaml --name "new-team" --conflict-strategy "merge" # 合并
   team-creator clone source.yaml --name "new-team" --conflict-strategy "force" # 覆盖
   ```

4. **配置导出功能细化** (优先级：低)
   - **问题**: F-012 功能描述较简略
   - **建议**: 在实施阶段补充详细设计
