## 📝 方案 B 详细内容

### 1. 任务状态管理 API
- `PUT /api/tasks/:taskId/status` - 更新任务状态
- `GET /api/tasks/overdue` - 获取延期任务列表
- `POST /api/tasks/notify` - 发送任务通知

### 2. 定时状态同步脚本
- 每 30 分钟检查一次 IN_PROGRESS 任务
- 超期 1 小时 → 提醒执行者
- 超期 4 小时 → 升级告警

### 3. 延期告警机制
- 任务预计完成时间前 1 小时提醒
- 超期后每 30 分钟提醒一次
- 严重超期 (4h+) 通知用户

### 4. DoD 检查清单
- 所有 Agent 的 AGENTS.md 添加检查清单
- "更新任务状态"作为必须项

### 5. Agent 轮询服务
- 每 5 分钟检查 inbox
- 发现新通知自动触发会话
