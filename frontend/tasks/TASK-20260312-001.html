🔧 改进方案

### 1. 任务状态管理模块

**文件**: `src/task-status-manager.js`

**功能**:

  • `updateStatus(taskId, newStatus)` - 更新任务状态
  • `checkOverdueTasks()` - 检查延期任务
  • `notifyAssignee(taskId)` - 通知执行者更新状态
  • `autoComplete(taskId)` - 自动标记完成 (当输出物都存在时)

    • **集成点**:

  • Agent 完成工作后调用 `updateStatus()`
  • 定时任务每 30 分钟调用 `checkOverdueTasks()`

    • ### 2. DoD 检查清单

    **位置**: 每个 Agent 的 `AGENTS.md`

    **清单内容**:

    ```markdown

    任务完成检查清单 (DoD)

  • [ ] 核心功能已实现
  • [ ] 自测通过
  • [ ] **任务状态已更新为 COMPLETED** ⚠️ 关键
  • [ ] 输出物已提交到指定位置
  • [ ] 相关文档已更新
  • [ ] 下游任务负责人已通知

    • ```

    ### 3. 定时状态同步

    **文件**: `scripts/task-sync.js`

    **功能**:

  • 每 30 分钟检查一次所有 IN_PROGRESS 任务
  • 超过预计时间 1 小时 → 发送提醒
  • 超过预计时间 4 小时 → 升级告警

    • ### 4. 测试验证

    **测试用例**:

  • 状态更新 API 测试
  • 延期检测测试
  • 告警通知测试
